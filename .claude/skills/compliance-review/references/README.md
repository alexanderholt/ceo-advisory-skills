# Compliance Review References

Place regulatory source documents in this folder. The /compliance-review
skill reads these files as its source of truth when conducting reviews.

## What to put here

- Relevant statutes and regulations (full text or key excerpts)
- Agency guidance documents (Dear Colleague Letters, FAQs, etc.)
- Industry standards and best practices documents
- Internal compliance policies

## Examples by industry

**Fintech / lending:** Federal and state lending regulations, consumer protection rules, agency guidance letters
**Healthcare:** HIPAA provisions, FDA guidance, state licensing requirements
**Education:** FERPA, accreditation standards, Department of Education guidance
**SaaS / data:** GDPR, CCPA/CPRA, SOC 2 requirements, data processing agreements

## Tips

- Name files descriptively: `hea-title-iv-part-g.md` not `doc1.md`
- Include the source and date in each file
- If more than 10 documents are present, the skill will ask which
  are most relevant to the specific review
